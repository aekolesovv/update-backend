# Swagger API Documentation

Эта папка содержит документацию Swagger для всех API роутов системы VSEBox.

## Структура документации

### Основные модули

- **`users.js`** - Документация для роутов пользователей
  - Регистрация и авторизация
  - Управление профилем
  - Смена пароля
  - Восстановление пароля

- **`products.js`** - Документация для роутов продуктов
  - Получение списка продуктов
  - Получение продукта по ID

- **`mailers.js`** - Документация для роутов почты
  - Отправка email сообщений

- **`auth.js`** - Документация для роутов аутентификации
  - CDEK API аутентификация

### Доставка и логистика

- **`delivery.js`** - Документация для роутов доставки
  - Создание заказов доставки
  - Получение статуса доставки

- **`calculators.js`** - Документация для калькулятора доставки
  - Расчет стоимости доставки

- **`countries.js`** - Документация для роутов стран и городов
  - Получение списка городов CDEK

### Платежи и финансы

- **`payments.js`** - Документация для роутов платежей
  - Создание платежей через Альфа-Банк
  - Проверка статуса платежей

- **`balance.js`** - Документация для роутов баланса
  - Управление балансом пользователя
  - Пополнение и снятие средств
  - Конвертация валют
  - История операций

### Управление данными

- **`orders.js`** - Документация для роутов заказов
  - Создание и управление заказами
  - Получение информации о заказах

- **`recipients.js`** - Документация для роутов получателей
  - Управление списком получателей

- **`addresses.js`** - Документация для роутов адресов
  - Управление адресами пользователя

### Администрирование

- **`admin.js`** - Документация для админ роутов
  - Управление пользователями
  - Административные функции

## Теги для группировки

Документация использует следующие теги для группировки API:

- `[Users]` - Пользователи и аутентификация
- `[Products]` - Товары и каталог
- `[Mailers]` - Почтовые сервисы
- `[Auth]` - Аутентификация внешних API
- `[Delivery]` - Доставка и логистика
- `[Calculators]` - Калькуляторы и расчеты
- `[Countries]` - Географические данные
- `[Payments]` - Платежные операции
- `[Balance]` - Управление балансом
- `[Orders]` - Заказы
- `[Recipients]` - Получатели
- `[Addresses]` - Адреса
- `[Admin]` - Административные функции

## Безопасность

Все защищенные роуты используют JWT токены через схему `bearerAuth`:

```yaml
security:
  - bearerAuth: []
```

## Форматы данных

- **Входные данные**: JSON в теле запроса
- **Выходные данные**: JSON в ответе
- **Кодировка**: UTF-8
- **Типы данных**: Строго типизированы согласно OpenAPI 3.0

## Примеры использования

Каждый роут содержит:
- Краткое описание (`summary`)
- Подробное описание (`description`)
- Параметры запроса (`parameters`)
- Тело запроса (`requestBody`)
- Схемы ответов (`responses`)
- Коды ошибок и их описания

## Обновление документации

При добавлении новых роутов:
1. Создайте новый файл в этой папке
2. Используйте стандартный формат JSDoc комментариев
3. Добавьте соответствующие теги для группировки
4. Обновите этот README файл

## Доступ к документации

Swagger UI доступен по адресу: `/api-docs` (настраивается в `app.js`)
